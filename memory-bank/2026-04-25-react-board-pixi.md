# 2026-04-25 React Board + Pixi Battlefield Update

## Summary

- The battle screen now renders through real React components instead of the old `renderBoard()` HTML string.
- Setup and mulligan screens still use the legacy string-template compatibility path.
- PixiJS receives `GameState` plus UI effect signals and renders only foreground battle effects.

## Files Added

- `src/components/react/CardView.tsx`
  - React hand card, minion card, persistent/trap card, keyword badges, and stat badges.
- `src/components/react/PlayerHUDView.tsx`
  - React player/enemy HUD with HP, mana, hand/deck counts, jump slot, and god-draw slot.
- `src/components/react/ReactBattleBoard.tsx`
  - React battle board composition, battle controls, pending choice modal, game-over modal, effect banner, and momentum panel.

## Files Updated

- `src/App.tsx`
  - Uses `ReactBattleBoard` for `state.screen === "game"`.
  - Keeps legacy `dangerouslySetInnerHTML` only for setup and mulligan.
  - Passes `GameState` into `PixiBattlefieldHost`.
- `src/game-view/pixi/PixiBattlefieldHost.tsx`
  - Accepts `state: GameState`.
  - Renders foreground attack trails, damage numbers, hit bursts, summon particles, and spell/trap bursts.

## Current Architecture Boundary

- React owns cards, HUD, board layout, buttons, modals, logs, and readable text.
- PixiJS owns only the non-interactive foreground battle-effect canvas.
- The removed battlefield underlay should stay removed until there is a deliberate board art/background pass.

## Pixi FX Follow-up

- `PixiBattlefieldHost` now receives `attackFx` and `cardFx` in addition to `GameState`.
- PixiJS owns the first wave of battle effects:
  - Attack trails from attacker to target.
  - Hit bursts at damaged heroes/minions.
  - Floating damage numbers generated by comparing previous and current HP/health snapshots.
  - Summon rings and particles for minion entry.
  - Spell/trap/projectile style bursts for `spellCast`, `trapTrigger`, and persistent/trap placement.
- The battle rules still live entirely in the engine/store. Pixi is display-only and does not decide damage or legal actions.
- The Pixi canvas is layered above the game surface with `pointer-events: none`; modal overlays remain above Pixi with a higher z-index.
- React battle cards no longer receive the old DOM animation flags for attack, impact, summon, or placement; React keeps selection/target state while Pixi owns the motion-heavy feedback.

## Pixi Coordinate Follow-up

- React battle entities now expose `data-pixi-entity-id`:
  - Hero HUDs: `P1_hero`, `P2_hero`.
  - Minions: their `instanceId`.
  - Persistent/trap cards: their `instanceId`.
- `PixiBattlefieldHost` now resolves effect origins and targets from real DOM element centers before falling back to estimated battlefield slot coordinates.
- Damage snapshots store the resolved point for each minion before state changes, so damage numbers and hit bursts still appear in the right place when a minion is destroyed and its DOM node disappears.
- Coordinate conversion accounts for the fixed Pixi host bounds and renderer scale, keeping effects aligned with the DOM layout across different viewport sizes.

## Battle Layout Follow-up

- The battle header is now a compact status strip instead of a three-column hero with embedded logs.
- The action log moved into the right sidebar below the momentum panel.
- The battle page now uses stable viewport rows:
  - Enemy field.
  - Player field.
  - Player hand.
- Card, minion, persistent, HUD, momentum, and log panels now have tighter height constraints and overflow behavior to prevent text from stretching the layout.
- Long card/HUD copy is clamped in CSS so the board remains scan-friendly during real play.
- The temporary Pixi battlefield underlay was removed after it visually polluted the battle layout.
- `PixiBattlefieldHost` now creates only one fixed `pixi-battlefield-effects` canvas above the React shell, with `pointer-events: none`.

## Card Detail Tooltip Follow-up

- `CardView` now exposes card inspection data for hand cards, board minions, persistents, and traps.
- Hand cards keep hover/focus inspection even when they are currently unplayable by using `aria-disabled` plus a guarded click handler.
- `ReactBattleBoard` owns one portal-based card detail tooltip and clears it when the hovered card disappears or a game action starts.
- The tooltip shows card name, type, cost, tags, stats, description, effect summaries, flavor text, and current status.
- `style.css` includes a fixed high-z-index tooltip layer so it appears above the Pixi effects canvas without blocking card clicks.
